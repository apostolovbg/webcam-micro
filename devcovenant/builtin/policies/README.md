## Overview
Builtin policies are shipped under `devcovenant/builtin/policies/`.
Each policy directory defines one policy contract and optional autofix/assets.

Runtime behavior is sourced from AGENTS policy definitions generated by
refresh from policy descriptors plus metadata resolution layers.

## Layout Contract
Each policy directory uses this layout:
- `<policy-id>.yaml` descriptor
- `<policy-id>.py` runtime check script
- optional `autofix/*.py`
- optional `assets/` templates

The directory name and script/descriptor basename must match policy ID.

## Descriptor Contract
Descriptors define:
- `id`
- `text` policy prose
- `metadata` defaults

Resolved metadata is computed by runtime precedence and then written into:
- `devcovenant/registry/registry.yaml`
- AGENTS managed policy block

Example metadata-owned runtime behavior:
- `workflow_runs` are resolved from the active profile stack, recorded in
  the workflow contract, and executed through the core-owned `run`
  command surface.

## Policy Runtime Contract
Policy scripts expose policy checks and return structured violations consumed
by engine formatting and gate behavior.

Policies should remain language-agnostic. Language-aware analysis is delegated
to shared translator runtime, which returns normalized language units.

## Autofix Contract
Autofix helpers are optional.
When present, they live in `autofix/` and follow language-aware default
resolution rules:
- language-specific autofix helper when available
- `autofix/global.py` as the global default otherwise

A custom policy override suppresses builtin autofix helpers for that policy ID.

## Workflow
1. Update descriptor and script together.
2. Update mirrored tests under `tests/devcovenant/builtin/policies/...`.
3. Keep tests behavioral/contract-focused; do not add placeholder stubs.
4. Run `devcovenant refresh` after descriptor metadata changes.
5. Run full gate sequence before finalizing:
   `gate --start` -> `gate --mid` (rerun until clean) ->
   `run` -> `gate --end`.
